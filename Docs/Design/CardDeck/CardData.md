# 카드 데이터 설계

## 개요

카드의 시스템 내부 데이터 구조를 정의한다. 기획 관점의 카드 설명은 [카드 시스템](Card.md)을 참고.

---

## 1. 공통 요소

모든 카드가 공유하는 기본 요소.

| 요소 | 설명 |
|------|------|
| 고유 ID | 카드를 식별하는 고유값 |
| 이름 | 카드 이름 |
| 분류 | 유닛/스펠/저주 |
| 등급 | 일반/희귀/서사/저주 |
| 마나 비용 | 사용에 필요한 마나 |
| 키워드 | 선봉, 붕괴 등 (복수 가능) |
| 강화 카드 | 강화 시 변환되는 카드의 ID (없으면 강화 불가) |
| 효과 설명 | 카드의 능력 텍스트 |
| 일러스트 | 카드 이미지 |

- 강화된 카드는 별도 카드로 존재한다 (예: "보병"과 "보병+"는 각각 독립된 카드)
- 강화 카드 요소가 다음 카드를 가리키는 체인 구조 (예: 보병 → 보병+ → 보병+2)
- 잠재력 키워드가 없는 카드는 1단계 강화 카드까지만 존재

| 요소 | 설명 |
|------|------|
| 소환 유닛 | 소환할 [유닛](../Combat/Unit.md) 참조 |

### 2.2 스펠 카드

| 요소 | 설명 |
|------|------|
| 효과 타입 | 데미지/회복/버프 등 |
| 대상 | 단일/범위/전체 |
| 사거리 | 영웅 기준 효과 적용 거리 (공격 계열 스펠) |
| 수치 | 효과 수치 |

### 2.3 저주 카드

| 요소 | 설명 |
|------|------|
| 패널티 | 패널티 효과 |
