# 문서 작성 가이드

## 개요

프로젝트 문서의 일관성을 유지하기 위한 작성 규칙을 정의한다.

---

## 1. 문서 위치

| 문서 종류 | 위치 | 예시 |
|-----------|------|------|
| 시스템/파트 기획 | `Docs/Design/` | Combat/CombatSystem.md, CardDeck/Card.md |
| 개발 가이드 | `Docs/Guide/` | Architecture.md |
| 프로젝트 전체 | `Docs/` | Overview.md, Roadmap.md |

---

## 2. 디자인 문서 권장 구성

Design 폴더의 기획 문서는 아래 구성을 권장한다. 섹션마다 경우에 따라 생략/추가 가능.

| 섹션 | 내용 |
|------|------|
| **개요** | 이 시스템이 무엇인지, 어떤 역할인지 한두 줄 요약 |
| **관련 문서** | 이 시스템과 연관된 다른 Design/Guide 문서 링크 |
| **구성 요소** | 시스템을 이루는 요소 정의 (스탯, 타입, 분류, 파라미터 등) |
| **동작** | 요소들이 어떻게 작동하는지 흐름/규칙 설명 |
| **기타 사항** | 예외 케이스, 밸런스 고려, 추후 확장 등 |

---
