# Deck Defence - 개발 로드맵

## 마일스톤 개요

| 단계 | 목표 |
|------|------|
| **M0: 기획** | 기획 문서 완성 |
| **M1: 프로토타입** | 핵심 루프 검증 |
| **M2: 알파** | 1권역 플레이 가능 |
| **M3: 베타** | 전체 콘텐츠 |

## M0: 기획

> 목표: 게임의 전체 방향성과 시스템 설계를 문서화한다. 개발 진행에 따라 지속적으로 갱신.

- 프로젝트 오버뷰 및 핵심 시스템 기획서 작성
- 개발 가이드 작성
- 기획 리뷰 및 보완

---

## M1: 프로토타입

> 목표: 핵심 게임 루프(카드 사용 → 유닛 소환 → 실시간 전투)가 재미있는지 검증한다.

### M1.1 프로젝트 세팅

- [ ] Unity 프로젝트 초기 구성
  - 폴더 구조, 기본 씬 구성
  - 아트 리소스 임포트 및 정리
  - 기본 카메라 및 해상도 설정

### M1.2 전투 필드

- [ ] 사이드스크롤 전투 필드 구현
  - 배경 레이어 구성
  - 아군/적군 진영 영역 정의
  - 유닛 이동 경로 및 충돌 영역

### M1.3 유닛 시스템

- [ ] 기본 유닛 구현
  - 유닛 스탯 (HP, 공격력, 공격속도, 이동속도)
  - 소환 → 이동 → 교전 → 사망 생명주기
  - 자동 전투 (가장 가까운 적 타겟팅, 기본 공격)
  - 유닛 애니메이션 연동 (Idle, Walk, Attack, Death)

### M1.4 카드 & 마나 시스템

- [ ] 마나 시스템 구현
  - 시간 경과에 따른 마나 자동 회복
  - 마나 UI 표시
- [ ] 카드 핸드 시스템 구현
  - 덱 → 핸드 드로우
  - 카드 UI (코스트, 이름, 일러스트)
  - 카드 사용 (마나 소모 → 유닛 소환)
  - 핸드 리필 로직

### M1.5 적 시스템

- [ ] 기본 적 스폰 구현
  - 웨이브 기반 스폰 로직
  - 적 유닛 AI (전진 → 교전)
  - 스폰 테이블 (타이밍, 구성)

### M1.6 영웅 시스템

- [ ] 영웅 필드 배치 구현
  - 영웅 필드 위치 및 존재
  - 영웅 HP (= 패배 조건)
  - 영웅 기본 행동

### M1.7 전투 UI

- [ ] 전투 화면 UI 구현
  - 마나 바
  - 카드 핸드 영역
  - 영웅 HP 바
  - 웨이브 진행 표시
  - 전투 결과 (승리/패배) 화면

### M1.8 통합 & 검증

- [ ] 전체 전투 루프 통합 테스트
  - 전투 시작 → 카드 드로우 → 마나 관리 → 유닛 소환 → 전투 → 승/패 판정
  - 밸런스 초안 조정 (마나 회복 속도, 카드 코스트, 적 스폰 간격)

**검증 항목**
- 실시간 카드 사용 + 사이드스크롤 디펜스 조합이 재미있는가?
- 마나 관리와 카드 타이밍 판단에 전략적 깊이가 있는가?
- 전투 페이스가 적절한가?

## M2: 알파

> 목표: 1권역을 처음부터 끝까지 플레이할 수 있는 빌드를 만든다.

### M2.1 캠페인 구조

- [ ] 분기형 맵 탐색 구현
  - 맵 생성 (노드, 경로, 분기)
  - 맵 UI 및 노드 선택
  - 노드 타입 (전투, 상점, 휴식, 이벤트, 엘리트, 보스)

### M2.2 덱빌딩

- [ ] 덱 관리 시스템 구현
  - 카드 획득 (전투 보상)
  - 카드 제거
  - 카드 강화
  - 덱 확인 UI

### M2.3 캠페인 시스템

- [ ] 상점 시스템 구현
  - 카드/유물 구매, 카드 제거 서비스
- [ ] 유물 시스템 구현
  - 유물 획득 및 패시브 효과 적용
- [ ] 휴식 노드 구현
- [ ] 기본 이벤트 구현

### M2.4 전투 확장

- [ ] 적 다양화
  - 일반/엘리트/보스 적 구분
  - 적 특수 능력
- [ ] 카드 다양화
  - 유닛 카드 외 스펠 카드 등
- [ ] 영웅 1종 완성
  - 고유 능력 및 초기 덱 구성

### M2.5 UI 통합

- [ ] 전체 게임 플로우 UI
  - 캠페인 시작 → 맵 → 전투/상점/이벤트 → 보스 → 클리어/사망

- [ ] 세이브/로드 구현
